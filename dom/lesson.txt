DOM - document object model (объектная модель документа) - дерево объектов в памяти(оперативной) которое содержит все что на странице

DOM интерфейс преобразует все что есть в хтмл коде в ноды(объекты) и их есть 12 типов

Обязательно DEFER должен быть в корне прикрепленного скрипта! без него не выводятся (иногда при определенных условиях) данные к которым мы обращаемся!

children - свойство которое можно использовать для того чтобы добраться до нужного нода (только когда мы на 1000% уверены что тип разметки меняться не будет)

.closest()
document.querySelector("#inputComment"); - для одного элемента
document.querySelectorAll(); - для выбора нескольких элементов (при выборе прилетит псевдо-массив)
Array.from преобразует псевдо массив для работы с ним его стандартными методами

Ориентироваться на имена свойств а не атрибутов при работе с ними

основные свойства

.className
.classList
.value - не меняет хтмл код
.dataset - позволяте получать данные из атрибутов
.style помимо стилей может задавать динамически координаты
.attributes

методы

.textContent - можно даже записать используя его(при вызове дает строку без тегов, только текст) get/set
.innerHTML - get/set - тулит в хтмл код то что мы добавили + парсит тэги + можно хуйнуть тег вместе используя строку(так как он возвращает строку) '<strong></strong>' по сути рендеря то что вы добавляете в ТЭГ!

.append - добавляем в конец
.prepend - добавляем в 
они создают или переносят но не копируют
